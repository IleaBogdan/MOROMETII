## Overview

**Interventii24** is intended to provide a structured, reliable platform for managing interventions (e.g., service calls, maintenance tasks, or incident responses) in a centralized way.

- **Purpose**: Handling of frequent overloading of emergency services in cities, reducing delays that lead to preventable deaths, longer hospitalizations, and high acute-care costs
- **Target users**: Volunteers as a userbase for our application; Hospital managers as potential customers for our system implementation
- **Key capabilities**:
  - Creation and tracking of interventions by dispatch officers
  - Assignment of interventions to responsible personnel
  - Status monitoring and history of interventions
  - Basic reporting or analytics

### Languages

- **ReactNative with TypeScript** – main frontend implementation
- **C#** – backend server
- **SQL Server** – smaller scripts and tooling

### Typical Components

- **Frontend**: TypeScript-based SPA framework; ReactNative
- **Backend**: ASP.NET Core Web API (C#)
- **Database**: SQL Server
- **Build & Tooling**:
  - Node.js `npm`
  - .NET SDK
